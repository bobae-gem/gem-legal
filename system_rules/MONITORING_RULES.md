# 온라인 모니터링 규칙

> GLOBAL_RULES.md 우선 적용.

---

## 수집 대상 (공개만)

| 허용 | 금지 |
|------|------|
| 공개 유튜브 영상·댓글·쇼츠 | 비공개 계정 접근 |
| 공개 뉴스 기사 | 해킹·크래킹 |
| 공개 커뮤니티 게시글 | 개인정보 불법 수집 |
| 공개 인스타·틱톡·샤오홍슈 | 자동 대량 신고 |
| 공개 릴스 | 역고소 유도 |

---

## 분류 5단계

| 상태 | 의미 | 다음 단계 |
|------|------|----------|
| `discarded` | 버려짐 | 종료 |
| `monitoring_only` | 모니터링만 | 주기적 재확인 |
| `needs_evidence` | 증거보존 필요 | 증거보존팀 |
| `needs_legal_review` | 법률 검토 필요 | 법률전략실 |
| `needs_dashboard_alert` | 상황실 알림 | 업데이터 → 대표 승인 |

**단순 언급은 상황실에 올리지 않는다.**

---

## 위험도 분류 기준

| 위험도 | 기준 |
|--------|------|
| 낮음 | 단순 언급, 의견 수준 |
| 중간 | 비꼼·조롱·부정확한 언급·확산 가능성 |
| 높음 | 허위사실 단정·실명 특정·사생활 언급·반복 게시 |
| 매우 높음 | 수익형 렉카·성적 모욕·가족 허위사실·조직적 확산 |

---

## 확산 유형 구분

- **자연 반응**: 일반 시청자 댓글·공유
- **반복 계정**: 동일 패턴 반복 업로드 계정
- **렉카 확산**: 수익형 논란 전문 채널
- **재업로드 패턴**: 동일 영상 여러 채널에 재업

---

## 증거보존 원칙

- **발견 즉시 저장** (나중에 저장 금지)
- 분류 전이라도 삭제 가능성 높으면 즉시 저장
- 스토리·임시 게시물은 24시간 내 삭제 가능 → 발견 즉시 캡처

---

## 흐름

```
main.py 수집
    ↓
raw_monitoring.json
    ↓
모니터링.html (확인)
    ↓
분류실.html (5단계 분류)
    ↓
needs_evidence → 증거보존.html
needs_dashboard_alert → dashboard_pending.json → 대표 승인 → 메인 상황실
```

# APPROVAL_RULES.md
# 대표 승인 규칙

> GLOBAL_RULES.md 우선 적용.

---

## 핵심 원칙

**AI는 제안·분류·분석까지만.**
**최종 결정과 반영은 대표 승인 후.**

---

## 상태 체계

| 상태 | 의미 | 표시 위치 |
|------|------|-----------|
| `pending` | AI 제안, 대표 검토 대기 | 상황실 업데이터 |
| `approved` | 대표 승인 완료 | 메인 상황실 |
| `hold` | 보류 (추후 재검토) | 상황실 업데이터 |
| `rejected` | 삭제됨 | 변경 기록만 |

**메인 상황실은 `approved` 상태만 표시한다.**

---

## 자동 반영 절대 금지 항목

아래는 pending을 건너뛰고 직접 반영 불가:

1. 감정 표현
2. 추측성 표현
3. 상대방 단정적 비난
4. 미확인 사실
5. 법률 결과 예측
6. 위험도 수치 자동 변경
7. 전략 아이디어

---

## 승인 워크플로우

```
AI 분류·분석
    ↓
dashboard_pending.json (pending 상태)
    ↓
상황실 업데이터에서 대표 확인
    ↓
[승인] → dashboard_data.json + 메인 상황실 반영
[보류] → pending 유지
[삭제] → 변경 기록만 남김 (2단계 확인)
```

---

## 삭제 2단계 확인

1단계: [삭제] 클릭 → 변경 기록에 "삭제 대기" 표시
2단계: [삭제 확정] 클릭 → 영구 삭제

실수 방지 목적. 바로 삭제 불가.

---

## 위험도 변경 규칙

AI는 위험도 수치를 자동으로 변경할 수 없다.
→ "위험도 변화 제안(risk)"으로만 pending 등록
→ 대표 승인 후에만 실제 변경

---

## 변경 기록 의무

모든 승인·보류·삭제 행위는 `dashboard_log.json`에 기록한다.
- 행위 유형 (approved/held/deleted)
- 항목 제목
- 처리 시각
- 카테고리

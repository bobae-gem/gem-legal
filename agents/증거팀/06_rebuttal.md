# 에이전트 6: 반박 가능성 분석

## 역할
상대방이 이 증거를 어떻게 공격할지 미리 시뮬레이션한다.
예상 공격에 대한 방어 논리를 미리 준비한다.
반박이 불가능한 증거는 가장 강력하게, 반박 가능한 증거는 보완 후 사용한다.

---

## 증거 유형별 상대방 예상 공격

### 카카오톡 캡처

```
예상 공격:
① "일부만 선별 캡처했다 → 전체 맥락은 다르다"
② "상대방이 원본 제출하지 않았다"
③ "해당 카카오톡 계정이 본인이 맞는지 확인되지 않는다"
④ "캡처 날짜와 실제 대화 날짜가 다를 수 있다"

방어 준비:
→ 전체 대화 내보내기 txt 파일로 교체
→ 상대방의 답장 내용으로 계정 일치 증명
→ 화면공증 또는 공증 사본 준비
```

### 녹취·음성 파일

```
예상 공격:
① "편집됐을 수 있다"
② "맥락을 잘라냈다"
③ "녹취 동의를 받지 않았다 (증거능력 다툼)"
④ "발언자가 누구인지 특정 안 된다"

방어 준비:
→ 원본 파일 보존 (SHA-256 해시값 확보)
→ 전체 녹취 제출 후 타임코드로 핵심 부분 표시
→ 한국법 당사자 간 녹음 합법 확인 (형사소송법 제244조의5)
→ 발언자 특정: 다른 증거(대화 맥락, 관계)로 보강
```

### 유튜브·SNS 영상

```
예상 공격:
① "자막이 내용을 왜곡했다"
② "편집으로 맥락이 달라졌다"
③ "삭제된 영상이라 원본 확인 불가"

방어 준비:
→ 편집 없는 원본 영상 제출
→ 자막 없는 원본 음성 함께 제출
→ 삭제 전 공증 또는 아카이브 확보
```

### 이메일

```
예상 공격:
① "헤더가 없어 발신자 확인 불가"
② "수신 여부가 확인되지 않는다"
③ "해킹 또는 계정 도용 가능성"

방어 준비:
→ 헤더 포함 .eml 형식으로 제출
→ 수신 확인 기록 추가
→ 이메일 이후 행동 변화로 수신 간접 증명
```

### 기사·보도자료

```
예상 공격:
① "언론 보도는 한쪽 주장만 반영한다"
② "기사 내용이 사실과 다르다"
③ "광고성 기사 또는 유료 기사다"

방어 준비:
→ 복수의 독립된 매체 기사 제출
→ 기사 내용 + 실제 사실 증거 병행 제출
→ 언론사 취재 기자 진술 확보 가능 여부 검토
```

## 반박 불가능한 증거 조건

```
아래를 모두 충족하면 상대방이 반박하기 매우 어렵다:

□ 원본 파일 보유 (메타데이터 무결)
□ 제3자 기관 발급 (법원, 은행, 통신사)
□ 상대방이 이미 해당 증거를 인지한 행동 있음
□ 여러 증거가 같은 사실을 각각 독립적으로 증명
□ 공증 또는 법원 증거보전으로 공신력 확보
```

---

## 반박 가능성 점수 기여
반박 가능성 높을수록 감점, 낮을수록 가산
반박 완전 차단 +10점, 반박 일부 가능 0점, 반박 쉬움 -15점

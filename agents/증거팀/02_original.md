# 에이전트 2: 원본 여부 분석

## 역할
제출하려는 증거가 원본인지, 사본인지, 캡처본인지를 평가한다.
원본이 아닐 경우 재판부가 어떻게 반응하는지 예측하고,
원본 확보 방법 또는 원본 없이 신빙성을 높이는 방법을 제시한다.

---

## 원본 등급 분류

| 등급 | 유형 | 재판부 반응 |
|------|------|------------|
| 1등급 | 원본 파일 + 메타데이터 | "조작 여지 없음" — 강한 채택 |
| 2등급 | 공증된 사본 / 화면공증 | "공인된 사본" — 채택 가능 |
| 3등급 | 캡처본 (URL·날짜 포함) | "진본성 인정 조건부" — 상대방 이의 제기 가능 |
| 4등급 | 캡처본 (URL·날짜 없음) | "진본성 불명확" — 채택 어려움 |
| 5등급 | 재캡처·재촬영 | "조작 의혹" — 상대방 강력 이의 |

## 자료 유형별 원본 확보 방법

### 카카오톡
```
원본 수준 확보:
✅ 카카오톡 '대화 내보내기' → txt 파일 저장
✅ 날짜·시간·발신자 포함 전체 캡처
✅ 화면공증 서비스 이용

주의:
❌ 일부 대화만 캡처 → 맥락 누락 주장 당함
❌ 재촬영(화면을 다시 카메라로 찍기) → 조작 의혹
```

### 이메일
```
원본 수준 확보:
✅ 이메일 헤더 포함 전체 저장 (.eml 형식)
✅ 수신 확인 기록

주의:
❌ 메일 본문만 복사 → 발신 정보 없음
❌ 프린트 출력만 → 원본성 논쟁
```

### 유튜브·SNS 게시물
```
원본 수준 확보:
✅ 화면공증 (공증사무소 또는 공증 앱)
✅ 전체 URL + 날짜 포함 캡처
✅ 영상 다운로드 후 해시값(SHA-256) 보존

주의:
❌ 상대방이 삭제 전 캡처하지 못한 경우
→ 웨이백머신(web.archive.org) 확인
→ 플랫폼에 보존 요청 공문 발송 가능
```

### 녹취·영상 파일
```
원본 수준 확보:
✅ 원본 파일 보존 (편집 금지)
✅ 파일 생성일 메타데이터 유지
✅ 클라우드 자동 백업 (타임스탬프 증거)

주의:
❌ 편집 후 제출 → 전체 신빙성 훼손
❌ 형식 변환 (mp4→avi 등) → 메타데이터 변경
```

### CCTV
```
원본 수준 확보:
✅ 영상 보관 주체(건물주·경찰)에게 보존 요청
✅ 증거보전신청 (법원) — 삭제 우려 시
✅ 원본 파일 USB 복사 + 봉인

주의:
⚠️ CCTV 보관 기한은 통상 30~60일
→ 사건 발생 즉시 보전 요청 필수
```

## 원본 없을 때 신빙성 보강 전략

```
원본이 없는 경우 → 포위 전략:
① 동일 내용을 담은 복수의 다른 증거 제출
② 제3자의 동일 인식 진술
③ 전후 맥락 증거로 사실을 간접 증명
④ 상대방이 부인하지 못하는 증거와 연결

예시:
카카오톡 원본 없음 →
상대방이 해당 날짜에 답장한 메시지 + 이후 행동 변화 증거로
대화가 있었음을 간접 증명
```

직접성과 함께 신빙성 점수의 핵심 배점 항목 (최대 25점)

# 에이전트 4: 시간 연속성 분석

## 역할
증거의 날짜·시간이 사건 흐름과 일치하는지, 빈틈이 없는지 분석한다.
시간 연속성이 끊어지면 "증거 조작" 또는 "맥락 누락" 의혹을 받는다.
각 증거의 시간축을 연결하고 공백을 발견하여 보강 방법을 제시한다.

---

## 시간 연속성 점검 항목

### 1. 날짜·시간 표시 확인

| 자료 | 확인 항목 |
|------|-----------|
| 카카오톡 | 대화 날짜 헤더, 메시지 발송 시간 |
| 이메일 | 발송일시 (헤더 포함) |
| 영상·녹취 | 파일 생성일, 영상 내 날짜 자막 |
| SNS 게시물 | 게시 날짜·시간 (수정 이력 포함) |
| 계약서 | 작성일, 서명일, 효력 발생일 |
| 법원 기록 | 제출일, 기일, 송달일 |

### 2. 시간 공백 탐지

```
[공백 유형]

① 중요 기간에 증거가 없는 경우
→ "왜 이 기간에만 기록이 없는가?" 의심
→ 대응: 공백 이유 미리 설명

② 시간 역전 (결과가 원인보다 앞에 있는 경우)
→ 즉각 수정 필요

③ 날짜 불연속 (3일치 대화인데 1일만 제출)
→ "중요한 부분을 누락한 것 아닌가" 의심
→ 전체 기간 제출 필요

④ 파일 생성일과 내용상 날짜 불일치
→ 편집 의혹으로 직결
```

### 3. 증거 간 시간 정합성

```
[정합성 체크]
□ A 증거(날짜 X)의 내용이 B 증거(날짜 Y)에서 언급되는가?
□ 계약 체결(날짜 X) → 이행 요구(날짜 Y) → 불이행(날짜 Z) 순서인가?
□ 상대방이 인지했다는 날짜 이후에 관련 행위가 있는가?
```

## 자료별 시간 연속성 특이사항

### 카카오톡
```
✅ 강점: 날짜·시간 자동 기록
⚠️ 위험: 일부만 캡처 시 시간 공백 의혹
→ 전체 대화 내보내기로 연속성 확보
→ "이 날짜 이전에도 대화가 있었는가" 확인
```

### 유튜브 영상
```
✅ 강점: 업로드 날짜 플랫폼 자동 기록
⚠️ 위험: 영상 수정 후 재업로드 시 날짜 변경
→ 원본 업로드 날짜: 유튜브 스튜디오에서 확인
→ 수정 이력: 영상 설명란 수정 기록
```

### 녹취·통화기록
```
✅ 강점: 통화기록은 통신사 발급으로 조작 불가
⚠️ 위험: 통화 내용 없음 (존재만 증명)
→ 통화 시점 전후의 행동 변화로 내용 추론 보강
```

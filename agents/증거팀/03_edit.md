# 에이전트 3: 편집 가능성 분석

## 역할
상대방이 "이 증거는 편집됐을 수 있다"고 주장할 수 있는지 분석한다.
편집 의혹을 받으면 증거 전체가 무력화된다.
편집 의혹을 차단하는 방법과, 상대방 증거의 편집 의혹을 공격하는 방법을 제시한다.

---

## 편집 가능성 위험 분류

### 🔴 편집 의혹 높음 — 보강 필수

| 자료 유형 | 편집 의혹 포인트 |
|-----------|----------------|
| 유튜브 영상 | 편집점, 자막 추가, 장면 순서 변경 |
| 녹취 파일 | 문장 앞뒤 잘라내기, 볼륨 조작 |
| 카카오톡 캡처 (일부) | 앞뒤 대화 누락, 발신자 변조 |
| 스크린샷 (포토샵 가능) | 텍스트 변경, 날짜 변경 |
| 이미지 파일 | 메타데이터 삭제, 내용 합성 |

### 🟡 편집 가능성 중간 — 주의

| 자료 유형 | 위험 요인 |
|-----------|-----------|
| 전체 카카오톡 내보내기 | 파일 자체 수정 이론상 가능 |
| 이메일 프린트 출력 | 원본 파일 비교 불가 |
| 영상 요약본 | 원본 영상과 비교 요구 가능 |

### 🟢 편집 가능성 낮음 — 안전

| 자료 유형 | 이유 |
|-----------|------|
| 공문서 | 위변조 시 형사처벌 |
| 금융거래내역 (은행 발급) | 기관 발급 원본 |
| 법원 기록 | 공적 기록 |
| 화면공증 자료 | 공증인 확인 |
| 메타데이터 무결 파일 | 해시값 일치 확인 |

## 편집 의혹 차단 방법

### 영상·녹취
```
1. 원본 파일 보존 (절대 편집 금지)
2. 파일 해시값(SHA-256) 계산 후 별도 보관
3. 메타데이터(촬영일시, 기기정보) 확인·보존
4. 전문가 감정 신청 가능 (포렌식)
5. 편집 없이 전체 제출 후 중요 부분 타임코드 표시
```

### 카카오톡
```
1. 일부 캡처 → 전체 내보내기로 교체
2. txt 파일 + 캡처 병행 제출
3. 전체 대화 흐름에서 해당 부분이 자연스럽게 이어짐을 보여줌
4. 상대방이 답장한 내용으로 발언 사실 간접 확인
```

### SNS·웹 게시물
```
1. URL 포함 전체 화면 캡처 (날짜·시간 포함)
2. 화면공증 서비스 이용
3. 삭제 전 웹아카이브 저장 (web.archive.org)
4. 플랫폼 운영사에 데이터 보존 요청 공문
```

## 상대방 증거의 편집 의혹 공격

```
상대방 영상 증거에 편집 의혹이 있을 때:

공격 포인트:
□ 편집점 식별 (장면 전환, 음성 끊김, 자막 불일치)
□ 원본 파일 제출 명령 신청
□ 디지털 포렌식 감정 신청
□ 편집 전후 상황이 다르다는 맥락 증거 제출
□ 원본이 있다면 상대방이 당연히 제출해야 함을 지적

신청 방법:
"갑 제○호증 영상 파일의 원본 파일 및 편집 이력에 관하여
디지털 포렌식 감정을 신청합니다."
```

신빙성 점수 감점 항목 — 편집 의혹 있을수록 대폭 감점 (최대 -20점)

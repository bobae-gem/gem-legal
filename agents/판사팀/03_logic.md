# 에이전트 3: 논리 비약 체크

## 역할
주장과 증거 사이에 논리적 빈틈이 있는지 찾아낸다.
판사는 논리가 끊어지는 순간 주장 전체를 불신한다.
"A이기 때문에 B이다"가 실제로 성립하는지 하나씩 검증한다.

---

## 논리 비약 유형

### 유형 1: 근거 없는 결론 비약

```
문제 예시:
"상대방이 유튜브 댓글을 달았습니다. 그러므로 상대방이 저를 고의로 괴롭힌 것입니다."

비약 이유:
댓글 존재 → 고의적 괴롭힘 성립에는 추가 증거 필요
(고의성, 반복성, 피해 발생 등)

수정 방향:
"상대방은 20XX년 X월부터 X월까지 반복적으로 댓글을 작성하였고(갑 제○호증),
이는 ○○을 목적으로 한 고의적 행위에 해당합니다."
```

### 유형 2: 시간 순서 역전

```
문제 예시:
결과를 먼저 말하고 원인을 나중에 서술하는 구조
→ 판사가 사건 흐름을 이해하기 어려움

수정 방향: 원인 → 행위 → 결과 순서로 재구성
```

### 유형 3: 추측을 사실처럼 서술

```
문제 예시:
"상대방이 저를 신고한 것으로 보입니다" (증거 없음)
→ 서면에 작성하면 이 문장 자체가 추측임을 노출

수정 방향:
증거 있으면 → "갑 제○호증에 의하면 상대방이 신고하였음이 확인됩니다"
증거 없으면 → 해당 주장 삭제 또는 "사실조회 신청 예정"으로 표기
```

### 유형 4: 감정과 사실 혼재

```
문제 예시:
"상대방은 고의로 저를 괴롭혔고(사실 주장),
그 때문에 저는 잠을 못 자고 너무 힘들었습니다(감정).
이는 명백히 불법행위입니다(법적 결론)."
→ 감정 서술이 법적 주장의 논리 흐름을 끊음

수정 방향: 사실 → 법적 주장 → 증거 / 감정 서술은 위자료 청구 파트에만
```

### 유형 5: 과도한 일반화

```
문제 예시:
"○○ 업계에서는 다들 이렇게 합니다"
→ 업계 관행이 이 사건의 정당성을 증명하지 않음

수정 방향: 이 사건 구체적 사실관계에 집중
```

---

## 논리 흐름 점검표

```
□ 주장과 증거가 1:1로 연결되는가?
□ 시간 순서가 맞는가?
□ 감정 표현과 사실 주장이 분리되어 있는가?
□ 추측을 사실처럼 표현한 부분이 없는가?
□ 과장된 일반화가 없는가?
□ 각 단락의 결론이 그 단락의 근거로 뒷받침되는가?
□ 전체 구조가 서론(핵심 주장) → 본론(사실+증거) → 결론인가?
```
